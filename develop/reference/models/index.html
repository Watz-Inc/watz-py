    models

    Data and response models returned from the client and those needed to pass into client methods.

    Activity

    Bases: BaseModel

    An activity of a subject.

    Attributes:

    Name Type Description
    uid str

    The activity's unique id.
    label str

    The activity's label, if no label was provided this will default to the uid.

    NewActivity

    Bases: BaseModel

    A creator model for an activity.

    Attributes:

    Name Type Description
    label str

    An optional label for the activity. If omitted, the label will default to the auto-generated uid.

    NewSubject

    Bases: BaseModel

    A creator model for a subject.

    Attributes:

    Name Type Description
    uid str

    The unique identifier for the subject. Currently, this must be a valid email address.

    NewTrace

    Bases: BaseModel

    A creator model for a trace.

    Attributes:

    Name Type Description
    uid str

    The trace's identifier, this must be unique to the subject/activity.
    data list[Any]

    The list of json-compatible data.

    RespCreateActivities

    Bases: RespBase

    POST: /api/v1/activities response model.

    Attributes:

    Name Type Description
    activities list[Activity]

    The created activity models, in the same order they were supplied.

    RespCreateSubjects

    Bases: RespBase

    POST: /api/v1/subjects response model.

    No specific attributes.

    RespCreateTraces

    Bases: RespBase

    POST: /api/v1/traces response model.

    No specific attributes.

    RespPing

    Bases: RespBase

    GET: /api/v1/ping response model.

    Attributes:

    Name Type Description
    status Literal['OK']

    Always "OK".
    whoami str

    The researcher's email address attached to the token.

    RespSubjects

    Bases: RespBase

    GET: /api/v1/subjects response model.

    Attributes:

    Name Type Description
    subjects list[Subject]

    The subjects of the researcher.

    RespTraceData

    Bases: RespBase

    GET: /api/v1/trace-data response model.

    Attributes:

    Name Type Description
    traces dict[str, dict[str, Optional[TraceData]]]

    dict[subject/act uid, dict[trace uid, TraceData]].

    RespTraceList

    Bases: RespBase

    GET: /api/v1/trace-list response model.

    Attributes:

    Name Type Description
    traces dict[str, dict[str, TraceMeta]]

    dict[subject/act uid, dict[trace uid, TraceMeta]].

    Subject

    Bases: BaseModel

    A subject of a researcher.

    Attributes:

    Name Type Description
    uid str

    The subject's unique id. Currently, this is always the subject's email address.
    activities list[Activity]

    The subject's activities.

    TraceData

    Bases: TraceMeta

    An extended TraceMeta object that contains the trace's data.

    Attributes:

    Name Type Description
    data list[Any]

    The list of json-compatible data.

    TraceMeta

    Bases: BaseModel

    A trace meta object. This is a lightweight object that contains the trace's metadata but excludes the data itself.

    Attributes:

    Name Type Description
    uid str

    The trace's unique identifier.
    length int

    The trace's length.
